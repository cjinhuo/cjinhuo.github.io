                        2021年（本命年）度总结

                        cjinhuo 2021-12-24 life

                        # 概要

                        这是一篇纯粹的个人年度总结，没有关于任何技术性的探究。

                        本文分成三个部分：

                        • 背景
                        • 2021年度总结
                        • 2022目标预定

                        # 背景

                        2021本命辛丑年即将过去，此时细细回忆今年的每一天，虽不是特出彩，但也有可圈可点，是总结也是反思，也是写给明年的自己。

                        # 2021年度总结

                        # 前端

                        • 学会了怎么架构一个迭代性良好的SDK，也在github开源了前端监控SDK，前前后后也有近1K的star（包括旧仓库），也有·十来家企业用了该SDK。
                        • 从中小厂跳入了大厂，也成功的从Vue转到了React，先前写React项目都是走马观花的浅尝辄止，现在要深入研究一番。

                        # 服务端

                        • 接触到了ES、rocketMq、sls、redis、mysql、nestjs等等node端的一些工具库， 也是用这些技术实现了前端监控的服务端，并在公司中落地以及覆盖率达到了95%以上
                        • 术业有专攻，上面这些技术也只是达到能用、会用的阶段，并没有持续深入下去，但是也足够撑住中小企业中的这种日均几十万的数据量。由于踩了许多坑，最终也学会了怎么构建一个良好的nestjs项目

                        # 升职加薪

                        • 意料之中，在年后的第二周参加了升职答辩，和组长排练了一晚的答辩PPT在答辩结束后收到一致好评，显而易见最终以压倒性的优势kill了其他升职候选人。

                        # 跳槽

                        • 既然升职，为何跳槽？
                        • 在毕业后的这一年多，很多人都说大厂至少要去一次，可以了解开发规范、学习内部基建产品顺便镀个金
                        • 跳槽想法的产生：如果你觉得在某家公司过于舒服，感觉每天的事情都在意料之中并不费吹灰之力便可以完成的话，大概率当前的你已经在「舒适区」中心了，五月份的我就是这种状态
                        • 走了就不能再和他们一起日常吃饭、吹水、健身，继续呆着每天无所事事，此时内心萦绕着我偶像「彭于晏」说过的一句话：「我还年轻吃苦趁现在」
                        • 2021字节两年经验前端面经

                        # 运动

                        • 骑行：五一劳动节和同事一起从「杭州市余杭区」骑到「湖州市德清县莫干山」，结果那边莫干山的民宿已经被订光了，那就去附近的「地理信息小镇」逛了一圈，来回将近90公里的骑行路程
                        • 骑行：国庆期间和前同事从「余杭区」骑到「青山湖」环岛，结果在距离家还有15公里的途中轮胎爆了，叫了辆货拉拉拖回去，来回60公里
                        • 骑行：从小区出发去「钱塘江大桥」环形，来回48公里

                        # 健身

                        从年初开始几乎每天下班后都去健身，效果也是蛮明显的，不仅仅是身材上改变，更多的是心理上的变化：从失眠到有困意、从躺尸到自己动手做饭、从普通到普信

                        # 吃一堑长一智

                        • 如果想持续维护开源项目的话，要是公司没有强制要求，不需要放入公司组下的Github

                        • 造轮子时技术选型尽量采用通用框架，开源时会更容易

                        • 在工作中永远也不要相信口头承诺，聊天记录或文档才是王道

                        • 面试的时候尽量低调，一切照常运行

                        # 正在学的技能点

                        # 打字

                        以前是用8个手指来写作、敲代码，小拇指几乎被我遗弃，导致打字速度和准确率有点差强人意，所以就在typingclub上面练习相对正确的打字方式，逐步开发小拇指的使用频率（曾经带过我的一位组长，看他打字就是一种享受）

                        # 英语

                        作为一个程序员，看英文文档是一件基本的事情，但是写英文文档就需要练习，讲口语更需要练习，所以目前也在往这方向靠拢，主要是方式：

                        • 日常英文文档阅读
                        • 看书：「英汉互译」、「单词15000」
                        • 写英文commit、英文文档、
                        • 计划报下线班（🌚待排期~）

                        # 语言的艺术

                        互联网公理：「需求是做不完的」，理解需求并用排出优先级，并利用专业知识来和产品、设计师进行battle，砍掉不必要的需求，尽量为自己多争取开发时间以及buffer
